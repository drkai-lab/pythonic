# pythonic

Строгие правила для современного Python 3.10+ (предпочтительно 3.12+). С акцентом на типизацию, стандартную библиотеку и код, который senior-разработчик примет на ревью без возражений. Создано, чтобы убить «на вид нормальный, но явно AI-шный» Python, который потом дорого обходится при поддержке.

## Установка

**Личная** (на машине, для всех проектов):

PowerShell:
```powershell
git clone https://github.com/drkai-lab/pythonic $env:USERPROFILE\.grok\skills\pythonic
```

bash / zsh:
```bash
git clone https://github.com/drkai-lab/pythonic ~/.grok/skills/pythonic
```

**В проекте** (в репозитории, чтобы вся команда работала по одним правилам):

```powershell
# PowerShell — отдельные команды, без &&
mkdir -p .grok\skills
git clone --depth 1 https://github.com/drkai-lab/pythonic .grok\skills\pythonic
```

На Unix просто меняй слеши. Ручной клон тоже работает.

Для других инструментов (Claude Code, Cursor и т.д.): `skill.md` — обычный текст. Правила и таблицу антипаттернов можно перенести вручную.

## Реальный пример до/после

Типичный код, который часто генерируют модели:
```python
def process(data):
    result = []
    for item in data:
        if item > 0:
            result.append(item * 2)
    return result
```

Pythonic-вариант:
```python
def process(items: list[float]) -> list[float]:
    return [x * 2 for x in items if x > 0]
```

Ещё больше (более 30) реальных примеров — в таблице антипаттернов в [skill.md](skill.md).

## Правила, которые реально enforced (выжимка)

- У всех функций и методов обязательна аннотация возвращаемого типа.
- Работа с путями — только `pathlib`. `os.path` под запретом.
- Только f-строки. `%` и `.format()` не принимаются.
- Для диспетчеризации по типу или структуре — `match`. Длинные цепочки `isinstance` считаются плохим запахом.
- Запрещены голые `except:`, изменяемые значения по умолчанию и `# type: ignore` без внятного обоснования.

Если версия Python не указана, по умолчанию используются идиомы 3.12+ и при необходимости уточняется целевая версия.

Полная версия (17 разделов, таблица фич по версиям Python и большая таблица антипаттернов) — в [skill.md](skill.md).

## Зачем это нужно

Python от LLM обычно выглядит прилично на первый взгляд. Через полгода поддержки почти всегда хочется переписать. Эта skill — накопленный опыт боли от реальных кодовых баз.

Сделано намеренно жёстко и коротко. Если нужны мягкие «best practices», берите другой skill.
